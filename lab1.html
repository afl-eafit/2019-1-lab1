Introduction

While regular expressions and finite automata describe languages differently, both actually represent the same type of languages, the "regular languages" [Hopcroft 2001]. In order to show this, it is first necessary to show that:

  • Every language defined by a finite automaton is also defined by a regular expression.
  • Every language defined by a regular expression is defined by a finite automaton.

A previous programming lab tackled the second statement. However in this programming lab the aim is to implement a program that shows the first one, and thus, showing the equivalency between finite automata and regular expressions.

Assignment (80%)

  • Download the GitHub repository 2018-2-lab1.

  • (70%) Using the data types in the above repository define a function:

    	  dfa2Regex :: DFA s c -> Regex c

    that takes a deterministic finite automaton and returns a regular expression that defines the same language. All tested DFAs will have their states numbered from 1.

Examples

Using the DFA.hs and Regex.hs modules in the above GitHub repository, this is what a sample input-output looks like:

      GHCi> dfa2Regex example0
      ∅

      GHCi> dfa2Regex example1
      ε

      GHCi> dfa2Regex example2
      (c+(b+a)b)(ab)*

      GHCi> dfa2Regex example3
      ε+1(01)*0+1(01)*

      GHCi> dfa2Regex example4
      (^o^)/*(-_-)
